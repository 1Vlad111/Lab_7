# ADR-003: Інтеграція платіжного шлюзу

## Статус
Accepted

## Контекст
Клієнтам інтернет-магазину (зокрема майстрам, які замовляють матеріали безпосередньо з будівельного об'єкта) потрібна можливість швидкої оплати в один клік зі смартфона. Введення довгих реквізитів банківської картки вручну призводить до відмов від покупки на фінальному етапі.

## Рішення
Інтеграція через LiqPay API з повною підтримкою Apple Pay та Google Pay.

## Обґрунтування
1. Це дозволяє реалізувати бізнес-можливість, чітко визначену в Customer Journey Map (ЛР №6), та підвищити загальну конверсію успішних оплат.
2. LiqPay є надійним рішенням на українському ринку з детальною документацією та готовими SDK.
3. Використання зовнішнього еквайрингу знімає з нашої системи відповідальність за зберігання та шифрування чутливих даних карток (відповідність суворим вимогам безпеки PCI DSS).

## Альтернативи
- Розробка власного платіжного рішення: Відхилено через величезні фінансові та часові витрати на сертифікацію безпеки.
- Інші міжнародні шлюзи (напр., Stripe/PayPal): Відхилено через потенційні обмеження або вищі комісії для локального українського бізнесу порівняно з LiqPay.

## Наслідки

### Позитивні
- Висока зручність для користувачів (можливість оплати в 1 клік).
- Високий рівень фінансової безпеки без навантаження на власні сервери.

### Негативні
- Повна залежність процесу покупок від стабільності зовнішнього сервісу (якщо API LiqPay недоступне, клієнти не зможуть здійснити оплату).
- Необхідність враховувати комісію платіжної системи у ціноутворенні магазину.
