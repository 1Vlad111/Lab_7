# ADR-002: Вибір архітектурного стилю

## Статус
Accepted

## Контекст
Необхідно забезпечити швидку розробку та легке розгортання інтернет-магазину "BudStore" на початковому етапі. Команда має обмежений час на запуск базової версії системи. Система повинна обробляти каталог товарів, кошик, замовлення та складські залишки як єдиний пов'язаний механізм.

## Рішення
Використовувати монолітну архітектуру (Monolithic Architecture).

## Обґрунтування
1. Для інтернет-магазину середнього розміру моноліт значно спрощує розробку та наскрізне тестування зв'язків між модулем продажів та складським обліком.
2. Не вимагає складного керування інфраструктурою мікросервісів та налаштування складної мережевої взаємодії між модулями.
3. Дозволяє швидко розгорнути проєкт на одному сервері без використання складних оркестраторів (на кшталт Kubernetes).

## Альтернативи
- Мікросервісна архітектура: Відхилено через надмірну складність для поточної стадії проєкту та відсутність потреби у незалежному масштабуванні десятків різних компонентів на етапі запуску.

## Наслідки

### Позитивні
- Швидкий старт розробки та просте локальне тестування.
- Простий та зрозумілий деплоймент (розгортання) на сервері.

### Негативні
- У майбутньому, при значному розширенні функціоналу та збільшенні навантаження, загальна кодова база може стати важкою для підтримки і потребуватиме рефакторингу.
