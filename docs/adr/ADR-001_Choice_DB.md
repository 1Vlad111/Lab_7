# ADR-001: Вибір СУБД PostgreSQL 16

## Статус
Accepted

## Контекст
Система інтернет-магазину "BudStore" вимагає високої надійності при проведенні операцій із замовленнями, оплатою та контролем складських залишків. Очікуване навантаження: до 3000 замовлень на добу та близько 150 одночасних користувачів у пікові години. Потрібна підтримка складних реляційних зв'язків між товарами, категоріями, характеристиками матеріалів, замовленнями та клієнтами.

## Рішення
Використовувати PostgreSQL 16 як основну реляційну СУБД.

## Обґрунтування
1. Повна відповідність ACID: Гарантує, що остання одиниця товару на складі не буде продана двом покупцям одночасно (цілісність залишків).
2. Підтримка JSONB: Зручно для зберігання специфічних технічних характеристик будматеріалів (розміри, склад, вага), які можуть відрізнятися для різних категорій.
3. Висока продуктивність: Швидка робота складних запитів при фільтрації товарів за ціною, брендом та параметрами.

## Альтернативи
- MySQL: Відхилено через менш досконалу систему обробки складних аналітичних запитів для звітності.
- MongoDB: Відхилено, оскільки дані магазину (замовлення, оплати) мають жорстку структуру та вимагають складних JOIN-операцій.

## Наслідки

### Позитивні
- Надійна робота з фінансовими даними та складом.
- Висока розширюваність структури каталогу товарів.

### Негативні
- Необхідність налаштування регулярного бекапування для збереження історії замовлень.
